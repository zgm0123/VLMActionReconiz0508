# MiniCPM-V-4.5 视频推理：提示词来源与结果输出说明

对应脚本：`minicpm_native/infer_video.py`

---

## 一、提示词描述（Action Descriptions）从哪来

### 1. 加载入口

`build_prompt()` 函数在构造 prompt 时，会先调用 `load_action_descriptions()`：

```python
def load_action_descriptions():
    project_root = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
    desc_path = os.path.join(project_root, "action_descriptions.json")
```

**文件位置：**
```
VLM-Video-Action-Localization/
├── action_descriptions.json      ← 就是这个文件
├── minicpm_native/
│   └── infer_video.py
```

### 2. 使用条件

只有在命令行加了 `--detailed` 时，才会加载该 JSON 文件：

```bash
python infer_video.py --video xxx.mp4 --actions "追球,拉球" --detailed
```

- 不加 `--detailed` → 使用代码里**硬编码**的 `default_desc`
- 加了 `--detailed` 但 JSON 里没有该动作 → 同样回退到 `default_desc`

### 3. JSON 结构

每个动作包含 `start`（开始判定标准）和 `end`（结束判定标准）两个字段：

```json
{
  "追球": {
    "start": "判定目标：狗第一次表现出对球的兴趣并主动朝向球移动的瞬间...",
    "end": "判定目标：狗停止朝向球移动的第一帧..."
  },
  "犬拉": {
    "start": "判定目标：狗通过一根被人拉住的绳子，用力往后拉拽的第一帧...",
    "end": "判定目标：犬拉动作结束的第一帧..."
  }
}
```

当前支持的动作（JSON 中已定义）：

| 动作 | 说明 |
|------|------|
| 追球 | 狗奔跑追逐球/物体 |
| 犬追物体 | 狗奔跑追随人手中物体 |
| 犬尝试咬物体 | 狗张嘴试图咬住物体 |
| 犬咬住物体拉动 | 狗嘴里含物体并主动拉拽 |
| 犬咬住物体持续咬住 | 物体被狗嘴闭合包住的持住状态 |
| 犬咬 | 狗嘴里叼着球/物品（含跑动、甩头、咀嚼） |
| 拉球 | 狗通过被人拉住的绳子向后拽球 |
| 犬拉 | 狗通过被人拉住的绳子向后拉拽 |

## 二、结果输出路径与命名

### 1. 默认输出（不指定 `--output`）

```
VLM-Video-Action-Localization/
└── results/
    └── minicpm_v45/
        └── {video_name}_v45.json
```

- `{video_name}` = 视频文件名去掉扩展名
- 例如视频是 `ch1_20251120_132309short.mp4`，结果文件就是 `ch1_20251120_132309short_v45.json`

### 2. 自定义输出（指定 `--output`）

```bash
python infer_video.py --video xxx.mp4 --actions "追球" --output /path/to/result.json
```

### 3. 输出文件结构

```json
{
  "video": "/absolute/path/to/video.mp4",
  "model": "/absolute/path/to/MiniCPM-V-4_5",
  "fps": 3,
  "duration": 120.5,
  "segment_duration": 10,
  "actions": ["追球", "拉球"],
  "use_detailed": true,
  "raw_outputs": "=== 段 1 (0.0s-10.0s) ===\n[模型原始输出文本]...",
  "segment_results": [
    {"追球": {"start": 2.0, "end": 5.5}, "拉球": {"start": null, "end": null}},
    {"追球": {"start": null, "end": null}, "拉球": {"start": 12.0, "end": 14.5}}
  ]
}
```

### 4. 字段说明

| 字段 | 说明 |
|------|------|
| `video` | 输入视频的绝对路径 |
| `model` | 使用的模型路径 |
| `fps` | 采样帧率 |
| `duration` | 视频总时长（秒） |
| `segment_duration` | 分段时长（默认10秒） |
| `actions` | 分析的动作列表 |
| `use_detailed` | 是否使用了详细定义 |
| `raw_outputs` | 每段模型原始输出的文本拼接 |
| `segment_results` | 每段解析后的 JSON 结果数组，时间已做段内偏移修正（加上段起始时间） |

---

## 三、时间修正机制

模型输出的时间是**段内相对时间**（从 0 秒开始），`infer_video.py` 会自动加上段的实际起始时间：

```python
parsed[action]['start'] = round(parsed[action]['start'] + seg_start, 1)
parsed[action]['end']   = round(parsed[action]['end']   + seg_start, 1)
```

例如：第 2 段的起始时间是 10s，模型输出 `start=2.5`，最终结果就是 `12.5`。
